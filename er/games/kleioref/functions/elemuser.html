Die Funktion user

Die Funktion user hat folgende Form:

:user[schlüsselwort]

Sie bezieht sich auf das logische Objekt der Klasse user, das durch ein item-Kommando definiert sein muß. 

Eine solche Definition hat die folgende Form:

item name=anwender_gruppe_institution;usage=user;overwrite=yes
user password="einpasswort";type=group
exit name=anwender_gruppe_institution

Wenn unter kleio eine Funktion (z.B. eine katalogisierte Prozedur) und dabei die erste Stelle des Protokollcodes den Wert "1" annimmt, so müssen legale Userdaten (Name, Gruppe, Institution) sowie das Paßwort angegeben werden, damit die Funktion ausgeführt wird. Dass heißt, in der logischen Umwelt müssen ein entsprechender User und das zugehörige Paßwort bekannt sein. Dazu wiederum muß eine Definition vorliegen, die der obigen entspricht.

Die Funktion User wiederum schreibt die Zugangsdaten des jeweils  authentifizierten Users als Daten des Typs text aus der Umwelt heraus. Man benötigt dies, um wiederum (z.B. in JavaSkript-Anwendungen) eine Weitergabe der Authentifizierungsdaten durchführen zu können. Die Anwendung erfolgt ohne Schlüsselwort ist denkbar einfach:

write part=:user[]

Dies würde also in unserem Falle folgenden Wert zurückgeben: "anwender_gruppe_institution"

Daneben kann :user[] mit dem Schlüsselwort "passwd" aufgerufen werden. In diesem Falle wird das jeweilige Paßwort zurückgegeben. Der Befehl

write part=:user[passwd]

würde also den Wert "einpasswort" zurückgeben.
