Der Parameter package:


Der Parameter package erwartet als Parameterwert den Namen einer package -Vereinbarung, die zuvor durch den item -Befehl in die Umwelt eingeführt worden sein muß.

Der Parameter dient dazu die Auszeichnung vom XML-Dateien zu modifizieren.

Man habe zum Beispiel folgende einfache Markup-Datei:


<tag1>1234</tag1>
<tag2>Irgendein Text</tag2>
<tag3>Und noch ein Text</tag3>

Nun wollen wir aus einem bestimmten Grund alle drei tags jeweils als Elemente definieren und diese wiederum gemeinsam in einer Gruppe "dokument" ablegen. Zu diesem Zweck definieren wir eine package-Vereinbarung, die die Auszeichnungen (Markups) für die Gruppe "dokument" enthält:

item name=dok;usage=package;overwrite=yes
field start="<dokument>";
      limit="</dokument>"
exit name=dok

Das Paket mit dem Namen "dok" legt fest, daß jeweils an den Anfang bzw. das Ende der Daten der Tag <dokument> bzw. </dokument> eingefügt wird.

Für die oben aufgelisteten Daten definieren wir nun folgende Struktur:

database name=test;first=dokument;overwrite=yes;write=structure
part name=dokument;
     also=tag1,tag2,tag3
exit name=test

Die drei Markups "tag1", "tag2" und "tag3" sind also jetzt gemeinsam der Gruppe "dokument" zugeordnet. Gehen wir davon aus, daß die Rohdaten ein einer Datei mit dem Namen "datei.xml" enthalten seien. Damit können wir nun folgenden read-Befel formulieren


read name=test;other=markup;
                           origin="datei.xml";
                            repeat=yes;
                            package=dok


Entscheidend ist hier der letzte Parameter package mit dem Wert dok , der auf die oben definierte Verpackungsanweisung Bezug nimmt.

Die Rohdaten werden nun in dieser Form eingelesen:


<dokument>
<tag1>1234</tag1>
<tag2>Irgendein Text</tag2>
<tag3>Und noch ein Text</tag3>
</dokument>
