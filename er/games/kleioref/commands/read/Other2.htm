Der Parameter other


Der Parameter other legt legt das Format der Ausgangsdaten (Rohdaten) fest, die durch den read-Befehl in eine Kleio-Systemdatei umgewandelt werden sollen.

Gegenwärtig  kennt der Parameter vier verschiedene Ausgangsformate:

  • kleio: Dieser Parameter entspricht dem Standardverhalten des Systems. Er setzt Rohdaten voraus, die im "klassischen" Kleio-Format vorliegen,a lso eine ASCII-Datei, deren Inhalt durch die entsprechenden Datensignalzeichen für Kleio lesbar gemacht wurde.
  • dBase: Dieser Parameter ordnet an, daß dbf-Dateien, die durch die Datenbanksoftware dBase erzeugt wurde, in eine Kleio-Systemdatei überführt werden.
  • markup: Bei Angabe des Parameters markup setzt Kleio eine wohl geformte XML- oder SGML-Datei als Quelldatenformat voraus.
  • sisis: Der Parameter sisis ordnet die Verarbeitung von Daten im Format der Datenbanksoftware Sisis an. 

Ein Beispiel für den Einsatz der Parameters ist folgender read-Befehl:


read name=db1;other=markup;origin="rohdaten.xml"


Hier werden Daten aus der Datei "rohdaten.xml", die in einer Auszeichnungssprache vorligen (other=markup) in eine Systemdatei überführt, die unter dem Namen "db1" als Kleio-Datenbank ansprechbar ist.
