Der Befehl read


Der Befehl read dient dazu, Ausgangsdaten (Rohdaten) in eine Kleio-Systemdatei umzuwandeln.

Dabei bezieht sich der name-Parameter des read-Befehls immer auf den Namen einer Kleio-Datenbank, der wiederum im name-Parameter des database -Befehls inder zugehörigen Kleio-Strukturvereinbarung aufscheinen muß. Damit wird eine Verbindung zwischen den Rohdaten und der Strukturvereinbarung  hergestellt. Dies setzt implizit voraus, daß die Definition der Strukturvereinbarung der Syntax der Rohdaten entsprechen muß (bzw. vice versa).

Wenn die Syntax der Rohdaten von der Definition der Strukturvereinabrung abweicht, wird der read-Befehl eine entsprechende Fehlermeldung ausgeben.

Der read-Befehl steuert die Umwandlung der Rohdaten in eine Systemdatei in vielfältiger Weise. Unter anderem legt er das Format der Rohdaten wie auch den Pfad zu den Rohdaten fest. Der read-Befehl erlaubt bei Daten im Markup-Format in Verbindung mit einem package auch die Einfügung zusätzlicher Markups in die Daten.

Grundsätzlich kann ein read-Befehl unmittelbar Bestandteil der Datei sein, die die Rohdaten enthält.  In diesem Falle muß der Befehl den Daten voranstehen:

read name=fre
FAM$FRE-54-2 M$m/12.8.1759/Abt/Gottlieb;Christoph/Praezeptor,Paedagogium/Esslingen/led/16.1.1820/Esslingen
VM$m/Abt/Christoph;Michael/Schullehrer//1 MM$w/Abt/Tritschler/Maria;Margaretha///1
HEI$25.9.1792/Esslingen
F$w/18.1.1763/Albrecht/Johanna;Catharina;Christina;Eberhardina//Kirchheim/Wwe/12.5.1803/Esslingen
VF$m/Albrecht/Christian/Hofr Einnehmer/Kirchheim/1
MF$w/Albrecht/Bahnmaier/Christina;Catharina///1
EEP$m//Caspart//Pfarrer/Vaihingen wK$e/11.6.1783/m/Christian;Erhard
KH$FRE-54-187
In diesem Falle würde der Aufruf der entsprechenden Datei unter Kleio dazu führen, daß der read-Befehl ausgeführt wird und die nachfolgend vorkommenden Rohdaten in seine Systemdatei überführt werden.


Wenn die Rohdaten also in einer Datei mit dem Namen "rohdaten.dat" abgelegte wären, könnte der read-Befehl mit folgendem Aufruf gestartet werden:
kleiob rohdaten.dat
Ebenso könnte der read-Befehl getrennt von den Rohdaten in einer eigenen Datei abgelegt sein. In diesem Fall muß zwingend der Parameter origin angegeben werden, der dem system mitteilt, in welchem Pfad die Rohdaten zu finden sind, die durch den read-Befehl abgearbeitet werden sollen.
