Die Direktive user

Über die Direktive User können weitere Eigenschaften des Users festgelegt werden. Es gibt die Parameter

  • password

  • mail

  • date und

  • type



Der Parameter Password

Im Parameter passwort wird das Passwort für den Benutzer abgelegt. Der Wert ist als Zeichenkette in Hochkommata einzutragen.

Der Parameter mail

Mail speichert die e-Mail Adresse des Benutzers. Der Wert ist als Zeichenkette in Hochkommata einzutragen. Dieser Eintrag ist notwendig, um die Benachrichtiguzngsgsmachnismen des User-Kommandos nutzen zu können!

Der Parameter date

Hier wird das Datum definiert, zu dem ein Benutzeraccount ablaufen wird. Dies muss dem Format Tag. Monat. Jahr entsprechen.
Default: heute + 180 Tage

Der Parameter Type

Der Parameter type kann per Definiton drei Werte annehmen: domain, group oder user.
Ist ein Nutzeraccount mit type=domain eingerichtet, so ist es dem Benutzer erlaubt sowohl Gruppen von Nutzern als auch einzelne Nutzer anzulegen, die wiederum der Domäne zugehörig sind. Etwa kann der Admin eines Universitätsinstituts den Typen „domain“ aufweisen, um Gruppenverwalter oder Einzelnutzer anzulegen.

Der Eintrag hätte folgende Form:

item name=h.koeln_h.koeln_h.koeln; usage=user; explanations="h.koeln_h.koeln_h.koeln"

user password="xyz"; type=domain

project name="h.koeln"; overwrite=yes

exit name=h.koeln_h.koeln_h.koeln


Ein Benutzer der mit dem type group eingerichtet ist, kann hingegen nur einzelne Nutzer einrichten. Die wäre etwa ein Dozent in einem Seminar, der einzelnen Studenten Zugänge einräumt. In unserem Beispiel:

item name=i.sohn_i.sohn_h.koeln; usage=user; explanations="i.sohn_i.sohn_h.koeln"

user mail="i.sohn@xyz.de"; password="defghi"; type=group

project name="i.sohn"; overwrite=yes

exit name=i.sohn_i.sohn_h.koeln

Es handelt sich um einen Gruppenveralter innerhalb der Domäne „h.koeln“


Standardmäßig ist type als user voreingestellt.

Ein einfacher Nutzer kann seinerseits keine weiteren Nutzer mehr eintragen. In unserem Beispiel:

item name=j.nemitz_i.sohn_h.koeln; usage=user; explanations="j.nemitz_h.koeln_h.koeln"

user mail="jnemitz@dddd.de"; password="abcdef“;

project name="j.nemitz"; overwrite=yes

exit name=j.nemitz_i.sohn_h.koeln
