Das Objekt user

Ein item-Kommando mit dem Parameterwert user im Parameter Usage legt im Environment einen Nutzer an. Der Name der Vereinbarung entspricht hierbei dem Nutzernamen. Nutzername und Passwd (das in der Direktive user definiert wird), werden im Rahmen von cgi-Abfragen überprüft, wenn der Kleio-Protokollcode an der entsprechenden ersten Stelle auf „1“ gesetzt ist. Das heißt, der CGI-Aufruf, muß in diesem Falle auch einen Nutzernamen, der im entsprechenden Environment bekannt ist, und das zugehörige Passwd übergeben, damit die angeforderte stored procedure ausgeführt wird. Über die Direktiven user und project können weitere Eigenschaften des Benutzers wie sein Passwort und Gültigkeitsdauer des Accounts festgelegt werden:

Beispiel:
item name=testuser_diathek_k.dresden; usage=user; explanations="testuser_diathek_k.dresden"
user mail="test@t-online.de";date="21.11.2008"; password="xxxyyyzzz";
project name="k.dresden"; overwrite=yes
exit name=testuser_diathek_k.dresden
