Das Objekt procedure



Das Objekt der Klasse procedure erlaubt die Defintion sogenannter "katalogisierter Prozeduren". Darunter versteht man Kleio-Skripte, die unter einem symbolischen  Namen in die Umwelt einer Datenbank eingeführt werden. Die Skripte können unter Angabe des entsprechenden Namens aufgerufen und ausgeführt werden. Dies geschieht in lokalen Anwendungen durch den Befehl execute.
Wenn ein Webserver unter Kleio betrieben wird, kann eine katalogisierte Prozedur durch Angabe ihres Namens im entsprechenden CGI-Directory aufgerufen werden.
Im Aufruf einer katalogisierten Prozedur können dieser durch den Einsatz von Ersetzungsmarkern zugleich Variable übergeben werden, die entsprechend den Inhalt resp. das Ergebnis der entsprechenden Aufgabe steuern. Der Inhalt einer solchen Variable kann z.B. eine Abfragebedingung sein.
Grundsätzlich legt das logische Objekt den symbolischen Namen einer katalogisierten Prozedur fest. Daneben muß eine Quelle  benannt werden, in der der eigentliche Skriptcode der entsprechende Aufgabe zu finden ist.
 
Das folgende Beispiel entspricht der Definition der katalogisierten Prozedur "searchset":


item name=searchset;usage=procedure;overwrite=yes
catalogue source="datei.txt";type=file
exit name=searchset

Es wird also eine Prozedur mit dem Namen "searchset" angelegt. Der dazu gehörige Quellcode befindet sich in einer Datei mit dem Namen "datei.txt".
Die abweichende Benennung der Quelldatei erfolgt hier zur besseren Unterscheidung vom Namen der Prozedur. Realiter empfiehlt es sich, die den Namen der Prozedur und den Namen der Quelldatei - der Übersicht halber - gleiche Namen zu verwenden.
