Das Kommando item:

Das Kommando item dient dazu, eine Datenbankumgebung (Environment) zu definieren. Unter einer „Umgebung“ kann man einen Satz von Regeln verstehen, der sich auf das Handling der Daten bezieht. Während das eigentliche Data Dictionary im database-Befehl definiert wird, stellt der item-Befehl weitere Definitionen bereit. Diese werden in der Kleio-Terminologie auch als „logische Objekte“ bezeichnet.

Die Regelsätze eines item-Befehls beziehen sich nicht im spezifischen Sinne auf eine bestimmte Datenbank, sondern sie werden in der Umwelt der Datenbank bereit gehalten und können im Rahmen bestimmter Kommandos aufgerufen werden. Als „Umwelt“ kann hier standardmäßig das Directory gelten, in dem der jeweilige item-Befehl ausgeführt wird (vgl. hierzu den Parameter target).

Der Inhalt einer Datenbank wird durch die Umweltdefinitionen nicht verändert. Wenn also z.B. Ersetzungsregeln definiert werden, wie dies durch ein logisches Objekt der Klasse conversion möglich ist, so werden diese Ersetzungen lediglich auf die Daten angewendet. Die Ausgangsdaten bleiben davon aber unberührt und können jederzeit wiederum in der ursprünglichen Form verarbeitet werden.

Jeder item-Befehl eröffnet eine Umgebung, in der bestimmte Regeln vereinbart werden können, die sich auf die Handhabung der Daten beziehen. Die Grammatik dieser Regelns ist Abhängig vom Typ (also der Klasse) der Definition. Einstweilen wird hier nur die Definition der logischen Objekte der Klasse package und procedure näher beschrieben. Für die übrigen Definitionen gelten bis aus weiteres die Beschreibungen im Systemhandbuch.
