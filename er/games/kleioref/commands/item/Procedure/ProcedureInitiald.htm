Die Direktive initial

Die Direktive initial erlaubt es, einer katalogisierten Prozedur einen oder mehrere Ausgangswerte zuweisen.  Dies Initialisierung bezieht sich auf Prozeduren, denen durch Ersetzungsmarker variable Werte zugewiesen werden können, die das jeweilige Ergebnis der Prozedur bestimmen.
Man habe zum Beispiel eine Aufgabe, die aus mehreren Katalogen eine Liste erstellt, die die Auswahl eines einzelnen Katalog aus der Liste ermöglichen soll. Dabei wiederum soll einer der Kataloge als Defaultwert voreingestellt sein. Dazu bedient man sich folgender Aufgabe:

describe usage=catalogue;write=cgi;display=regular;first=@=cat2use@

Es wird hier - grob gesprochen - angeordnet, daß die Namen der in der Umwelt befindlichen Kataloge als Auswahlmeu ausgegeben werden sollen. Bemerkenswert ist hierbei der Parameter "first", der einen der Werte als "selected" markiert. Das heißt, daß dieser Wert im Auswahlmenu voreingestellt ist. Wie man sehen kann, wird hier nicht der Name eines bestimmten Katalogs eingetragen (obwohl dieses grundsätzlich auch möglich wäre), sondern ein Ersetzungsmarker gesetzt.
Das wiederum heißt, daß der Prozedur verschiedene Werte übergeben werden können, so daß je nach Auswahl ein anderer Wert an die Proezdur übergeben wird, der dann die Darstellung des Auswahlmenus bestimmt.
Daraus wiederum ergibt sich automatisch die Frage nach dem Startwert des Auswahlmenus.  

Dieser wird durch die Direktive initial festgelegt. Dabei bestimmt der Parameter name den Namen des Ersetzungsmarkers, dem ein Startwert übergeben werden soll, während der Parameter text den entsprechenden Wert als Konstante enthält.

In diese Fall hat die Definition der Prozedur als folgende Form:

item name=job1;usage=procedure;overwrite=yes
catalogue source="describe usage=catalogue;write=cgi;display=regular;first=@=cat2use@"
init name=cat2use;text="kuenstler"
exit name=job1
