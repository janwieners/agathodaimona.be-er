Das Objekt complex

Das Objekt der Klasse complex weist einzelnen oder Gruppen von Datenbanken einen symbolischen Namen zu. Der Name entspricht dem Wert des name-Parameters zum item-Kommando bei der jeweiligen Definition.  

Der zugewiesene Name kann durch ein describe-Kommando wiederum zurückgegeben werden. Ebenso können alle Namen, die Datenbanken zugewiesen wurden auf diese Weise abgerufen werden. Diese Namen wiederum können dann in anderem Kontext weiter verwendet werden. Typischerweise kann dies zum Beispiel im Rahmen eines environment-Kommandos geschehen.

Das heißt, es ist möglich durch das Objekt complex verschiedene Datenbanken nach bestimmten Kriterien zu gruppieren und anschließend in einer Auswertung durch Übergabe des Gruppennamens das Environment so zu verändern, daß einzelne Bestandteile (z.B. Kataloge) nur auf bestimmte Gruppen von Datenbanken referenzieren.

Man habe zum Beispiel verschiedene Datenbanken kulturhistorischen Inhalts. Eine Gruppe von Datenbanken gehört dem Fachgebiet "Archäologie" an, während die übrigen in einer Gruppe "bildende Kunst" zusammengefaßt werden sollen. Dies kann man folgendermaßen bewerkstelligen:

item name=archaeologie;usage=complex;overwrite=yes;
complex source=arachne,archgiessen
exit name=archaeologie

item name=kunst;usage=complex;overwrite=yes;
complex source=piransi,digidia,kunstmuc,karlsruhe
exit name=kunst

Es werden also hier die Datenbanken "arachne" und "archgiessen" in der Gruppe "archaeologie" zusammengefaßt, während analog die Datenbanken "piranesi", "digidia", "kunstmuc" und "karlsruhe" die Gruppe "kunst" bilden!

Ein entsprechender describe-Befehl zeigt dies an:

describe name=archaeologie;usage=complex;

Wiedergabe einer Vereinbarung aus Ihrer Bibliothek.
Name der logischen Einheit: archaeologie
Gattung der logischen Einheit: Datenbankkomplex
Erlaeuterungstext: arch

Datenbanken in diesem Komplex:
arachne
archgiessen

In einem nächsten Schritt erstellen wir mittels des "suspend/augment/complete"-Mechanismus einen Katalog "standorte", der für alle fünf Datenbanken, die Aufbewahrungsorte, der darin verwalteten Objekte enthält.

Im Rahmen einer Auswertung wollen wir nun alle Standorte abfragen, die mit "A" beginnen. Dabei wiederum soll ausschließlich auf die archäologischen Datenbanken zugegriffen werden.

Dazu wird vorab mittels des Kommandos environment die Umwelt so "zugeschnitten", daß nur mehr diejenigen Datenbanken berücksichtigt werden, die der Gruppe "archaeologie" angehören! Durch die Ausgabe des Datenbanknamens mittels der Funktion :database[] können wir dies prüfen:

environment complex=archaeologie
query name=kleio;part=/catalogue[standorte,complete,"a*"]
index part=:database[];maxi=1
stop


Die Ausfuehrung der von Ihnen gestellten Aufgabe beginnt.
arachne

archgiessen


Die Ausfuehrung der von Ihnen gestellten Aufgabe wurde beendet.

Auf Wiedersehen bei Ihrer naechsten Kleio Sitzung!

Das heißt, daß innerhalb des Katalogs "standorte" nur Einträge aus Datenbanken berücksichtigt wurden, die ihrerseits der Gruppe "archaeologie" angehören.
