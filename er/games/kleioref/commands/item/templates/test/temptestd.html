Die test-Direktive

Die Direktive test erlaubt temporäre Manipulationen an den Einträgen in Containern. Der Zweck ist folgender: Bei komplexen Servern kann (je nach Aufbau des Servers) der Eintrag einer Identifikationsnummer im Container vom Wert des entsprechenden Eintrags im zugehörigen Katalog abweichen. Dies ist. gewöhnlich die Folge von Zeichenkettenoperationen.

Beim Aufruf des entsprechenden Containers kann also der entsprechende Eintrag nicht im Katalog identifiziert und entsprechend nicht dargestellt werden. Dem Problem kann mit verschiedenen Maßnahmen beigekommen werden:

  1. Es ist möglich den Inhalt der betreffenden Information mit einem conditional Package zu verpacken . Dabei kann in der Bedingung eine Zeichenkettenoperation durchgeführt werden, die wiederum die Zeichenkette im Container so verändert, daß sie im Katalog aufgefunden werden kann.
  2. Die Direktive test erlaubt eine temporäre (nur adhoc wirksame) Manipulation der Zeichenkette im Container, um so sicherzustellen, daß diese im Katalog identifiziert werden kann.

Die Direktive kann mehrfach angegeben werden. In diesem Fall beziehen sich die Test jeweils immer auf die Ausgangszeichenkette, die somit in jeder Direktive identisch ist.
