Verwaltung von Containern


Unter einem "template" versteht man in Kleio ein System zur geordneten und optisch und funktional aufwendigen Wiedergabe von Daten, die ihrerseits in einer bestimmten festgelegten Form, sogenannten Containern vorliegen müssen.

Der gegenwärtige Einsatzzweck liegt in der Umsetzung der Container zu sortierten Arbeitsmappen und Präsentationen. Dabei werden in den Containern nicht die Bilddaten selbst (im Sinne von Objekten des Datentyps image) verwaltet, sondern Bildkennungen (Identifikationsnummern) des Typs text, die ihrerseits in einem Katalog enthalten sein müssen.

Der Aufruf eines Containers geschieht innerhalb eines Kleio-Kommandos. Da templates vor allem für interaktive Webanwendungen konzipiert sind, wird das ausführende Kommando meist in Form eeiner katalogisierten Prozedur, vorliegen. Das Komamndo bewirkt in groben Zügen folgendes:

Die Bildnummern werden dem Katalog der Bildnummern übergeben. Damit ist der jeweilige Bilddatensatz in der zugehörigen Datenbank eindeutig identifiziert. Es kann damit auch auf andere Informationen (z.B. Künstler, Titel etc.) zugegriffen werden. Zugleich kann unter Verwendung einer entsprechenden Package-Vereinbarung ein URL zu einer entsprechen Bilddatei auf einem Webserver dynamisch generiert werden, der es ermöglicht, das betreffende Bild in der Ausgabe darzustellen und ggf. "klickbar" mit weiteren Funktionen (z.B. Zoom-Funktion) zu versehen.

Ein Beispiel für die Verwendung eines templates ist folgender Kommandoblock:

@+ctype@:Folder@
@>ctype@supply@
describe name=@<user@;first=@=project@;usage=user;write=projects;write=cgi;
  template=desmap0; package=wrapfolderopenstart;display=wrapprojectopen
administer template=desmap0;
request=list;
user=@<user@;
project=@=project@;
password="@<pass@";
write=cgi;
package=wrapfolderopen;
display=wrapfoldermenu

Sowohl das Kommando describe als auch das Kommando administer in diesem Block verweisen mittels des jeweiligen Parameters template auf eine Vereinbarung der Klasse template.

Es ist die "Aufgabe" des Objekts template allgemeine Anweisungen bereit zu stellen, die das äußere Erscheinungsbild (i.e. das Design) der Arbeitsmappe bzw. der Präsentation festlegen. Dazu wiederum erfüllt das template grundsätzlich verschiedene Teilfunktionen:

  1. Das Template nimmt Bezug auf Vereinbarungen der Klasse package, die ihrerseits als allgemeine Anweisungen die äußere Form der Mappe bzw. der Präsentation festlegen. Salopp gesprochen, könnte man formulieren, daß das Template durch Bezug auf Package-Vereinabrungen einen allgemeinen Rahmen aus Anweiungen (i.d.R. HTML-Code) produziert, der um die URLs der Bilder herumgebaut wird und damit Aussehen (und Funktionen) der Mappen und Präsentation bestimmt. Der entsprechende HTML-Code kann seinerseits wiederum auf JavaScript-Programme verweisen, so daß das template sich als  ein Bestandteil eines insgesamt recht komplexen Zusammenhangs von Skript-Anweisungen darstellt. Die JavaScript-Programme können  weitere Funktionen wie etwa das Einfügen, Kopieren, Verschieben oder Löschen von Bildidentifikationen innerhalb der Container bereitstellen, so daß auf diese Weise die Reihenfolge der Bildwiedergabe innerhalb von Mappen (oder Präsentationen) variiert werden kann. 
  2. Die Bildnummern innerhalb der Container können über zugehörige Datenbanken eindeutig in den zugehörigen Datenbanken identifiziert werden. Das Template erlaubt die Zusammenfassung von Einzelinformationen dieser Datenbanken in bestimmten Variablen, die neben dem eigentlichen Bild ausgegeben werden können. Das heißt, es sit möglich neben oder anstelle der Darstellung eines Bildes auch Textinformation ausgeben zu können.

Es ist hier bereits mehrfach der Begriff des Containers gefallen. Aufgabe der Templates ist es, diese Minimalinformationen (üblicherweise im Rahmen einer katalogisierten Prozedur) in eine Website zu überführen.

Die Prozedur erfüllt dabei zwei Grundfunktionen:

  1. Die Bildnummer wird in einem Katalog von Bildnummern identifiziert und daraus einen URL zu einem Bild auf einem Webserver generiert
  2. Es werden Packages ausfgerufen, die das Bild in adäquater Form in Tabellen und Rahmen einpassen, so daß ein gestaltete Ausgabe entsteht.
