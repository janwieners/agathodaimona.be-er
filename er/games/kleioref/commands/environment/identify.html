Der Parameter identify

Der Parameter identify erwartet die Parameterwerte yes oder no. Der letztgenannte Wert ist das Standardverhalten des Systems.

Der Parameter bezieht sich auf den Aufruf von Kleio-Protokollcodes. Die erste Stelle eine Protokollcodes kann den Wert 1 oder 0 annehmen. Der Wert "0" verlangt keine Benutzeridentifikation, wogegen sich der Nutzer beim Wert "1" authentifizieren muß. 


http://prometheus.hki.uni-koeln.de/prometheus-cgi/kleioc/1010KlProm/exec/zoom/%223475%22/__kleioid/%22j%2euser4%5fj%2euse4r%5fh%2ekoeln%22/__kleiopass/%22passwd4%22


In diesem Aufruf wird also für den Benutzer "user4" das Paßwort "passwd4" verlangt. Durch Umstellung des Protokollcodes kann dieser Aufruf auch ohne Authentifizierung funktionieren:

http://prometheus.hki.uni-koeln.de/prometheus-cgi/kleioc/0010KlProm/exec/zoom/%223475%22

Dies entspricht damit dieser Funktion.

An dieser Stelle greift der Parameter identify. Wird der Wert "no" spezifiziert, so werden die Protokollcodes immer in der angegebenen Form abgearbeitet. Wird dagegen der Wert "yes" angegeben, so wird die erste Stelle der Protokollcodes immer auf den Wert "1" gesetzt, wodurch eine Authentifizierung zwingend notwendig wird.
