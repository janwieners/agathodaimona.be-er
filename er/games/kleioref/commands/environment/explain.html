Der Parameter explain

Der Parameter explain erwartet die Parameterwerte yes oder no. Der letztgenannte Wert ist das Standardverhalten des Systems.

Bei Angabe von explain=yes sucht das System bei der Initialisierung von Kleio in dem Directory, in dem die Systemkomponenten kl__.ms etc. liegen, bzw. im aktuellen Directory, nach einer Datei "explain.ini". Im Rahmen des Systemaufrufs sorgt der Informationsgehalt  dieser Datei dafür, daß eine anwenderspezifische Initialisierung vorgenommen wird. Es werden dadurch z.B. bestimmte logische Objekte in einem bestimmten Zustand bereit gestellt.

Der Parameter ist vor allem von Bedeutung, um im GUI einen bestimmten Zustand speichern und wieder laden zu können. Bis zur weiteren Entwicklung des GUI ist der Parameter damit weitgehend gegenstandslos. Eine Übertragung des Mechanismus auf WWW-Anwendungen ist in Vorbereitung.  

Der Parameterwert no unterdrückt eine anwenderspezifische Initialisierung.
