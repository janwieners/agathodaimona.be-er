Der Parameter part:


Der Parameter part erwartet als Parameterwert eine Pfadangabe, die zumindest ein Element umfassen muß. Es können verschiedene Elemente bzw. Pfadangaben durch Kommata getrennt angegeben werden. Dabei sind die Angaben hinsichtlich der Datenbankhierarchie zueinander relativ. Das heißt, das System befindet sich nach jedem Komma auf der Ebene der Hierarchie, die vor dem Komma erreicht war. Ggf. muß also durch Angabe von Informationsgruppen oder Gruppenfunktionen die hierarchische  Eeben gewechselt werden.

Beispiele für richtige write-Befehl sind die folgenden:

write part=:name
continuation

write part=:name,:vorname
continuation

write part=:name,:vorname,pdat:gebdat
continuation

write part=:name,:vorname,pdat:gebdat,root[0]:hko
continuation

write part=:name,:vorname,pdat:gebdat,query[]:hko
continuation

write part=:name,:collection[:vorname],pdat/kar:beruf,query[]:hko
continuation

write part=:each[]
continuation

write part=:total[]
continuation
