Der Parameter target:

Der Befehl write kann dazu eingesetzt werden, den gesamten Inhalt einer Datenbank in maschinenlesbarer Form auszugeben, also einen "Dump" zu erzeugen. Dazu dient der Parameter target. Er hat standardmäßig den Parameterwert human. Diese implizit eingestellte Angabe bewirkt eine Datenausgabe als lesbares Dokument. Die Ausgabeform läßt sich durch verschiedene weitere Parameter steuern.

Der Parameterwert machine bewirkt hingegen, daß der Gesamtinhalt der Datenbank als Datei im "klassischen" Kleio-Eingabeformat ausgegeben wird. Dabei werden die Felder durch Slashes (/) getrennt und die Feldnamen durch Etiketten explizit angegeben. Das Semikolon trennt Mehrfacheinträge. Originaltext- und Kommentarfelder werden durch die entsprechenden Sonderzeichen (% bzw. #) kenntlich gemacht.


Folgende Aufgabe

query name=basdat;part=per
write part=:total[];target=machine
stop


schreibt also den gesamten Inhalt der Datenbank Basdat als eine (nach klassischen Kleio-Konventionen) maschinenlesbare Datei heraus.

das Ergebnis hat also die folgende Form:

per $ idd = DOK030011 / name = ABELE / vorname = CARL / geschlecht = M / hko = KIRCHHEIM,TECK / land = OA KIRCHHEIM
pdat $ 
kar $ von-bis = 1.7.1846 - 1.7.1850 / beruf = FABRIKANT,SESSEL(CARL ABELE) # WSP
mob $
wsp $ name = ABELE / vorname = CARL / hko = KIRCHHEIM,TECK / land = OA KIRCHHEIM / beruf = FABRIKANT,SESSEL / id = DOK030011 /  verbleib = 1.000000 / ersteintrag = 1.7.1846 / letzteintrag = 1.7.1850 # AB 50 FORT
