Der Befehl write:

Das Kommandowort  write stellt einen der zentralen Ausgabebefehle unter Kleio dar. Write bewirkt die unsortierte Ausgabe des gesamten Inhalts einer Datenbank oder einer spezifizierten Teilmenge desselben. Die Reihenfolge der Datenausgabe entspricht der Reihenfolge, in der die jeweiligen Daten bei der Kompilation der Datenbank eingelesen wurden. Die Datenausgabe orientiert sich damit implizit an der internen Ordnungsnummer der Dokumente, wie sie durch die Elementarfunktion :order[] sichtbar gemacht werden kann.

Im Unterschied zum Befehl index werden unter write keine Ausgabespalten erzeugt. Vielmehr wird der gesamte Inhalt der angeforderten Daten in der Reihenfolge der Dateneingabe wiedergegeben. Dementsprechend darf der Parameter part ,  der die zur Ausgabe angeforderten Informationen anspricht, nur einmal angegeben werden. Die Pfadangaben und Elemente werden dann jeweils durch Kommata getrennt angegeben. Ein gültiger write-Befehl muß damit folgender Syntax entsprechen:

write part=:name,:vorname,/pdat:gebdat,/kar:beruf,root[0]:hko

Im Unterschied zum index -Befehl besteht im write Befehl eine Default-Einstellung hinsichtlich des Inhalts und Umfangs der Datenausgabe. Die bloße Angabe des Kommandowortes write entspricht daher einem voll gültigen Auswertungsbefehl. Ohne weitere Spezifikation gibt write den gesamten Inhalt der jeweils aktuellen Informationsgruppe sowie aller von dieser abhängigen Gruppen an. Der Befehl write ohne weitere Zusätze entspricht damit standardmäßig der expliziten Angabe

write part=:total[]

Im Unterschied dazu gibt

write part=:each[]

lediglich den Inhalt der aktuellen Gruppe aus, also jener Informationsgruppe, die im vorangegangenen  query -Befehl ausgewählt wurde.

Die Standardausgabe eines write-Befehls hat folgende Form:

per (123 = "DOK030011")

                beruf        FABRIKANT,SESSEL(CARL ABELE)

       mob (1 = "mob-1")

       wsp (1 = "wsp-1")
                name         ABELE

Wie erkenntlich werden nicht nur die Daten selbst, sondern weitere Zusatzinformationen ausgegeben. Diese umfassen den Namen der jeweiligen Informationsgruppe, die Ordnungsnummer der jeweiligen Gruppe (entspricht bei root[0] der Nummer des Dokuments, bei den abhängigen Gruppe der jeweiligen Ordnungsnummer bezogen auf die jeweils übergeordnete Gruppe) sowie  die Namen der Elemente. Neben den Kerninformationen werden auch die Aspekte Kommentar und Originaltext ausgegeben. Mehrfacheinträge (CARL;FRIEDRICH) werden jeweils auf einer neuen Zeile dargestellt. Durch entsprechende Einrückungen werden die hierarchischen Abhängigkeiten in der Datenbank ausgedrückt.

Umfang und Aussehen der Standardausgabe lassen sich durch verschiedene Parameter verändern. Insbesondere können die Angabe der Element- und Gruppennamen unterdrückt und die Ausgabe der Informationen ohne Umbrüche erzwungen werden.
