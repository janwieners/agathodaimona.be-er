Der Parameter bridge

Kleio erlaubt die dokumentenbezogene permanente Verknüpfungen von Datenbanken. Zwischen zwei Datenbanken können so Verweise angelegt werden, die der Logik folgen, daß in Datenbank  1 ein Element des Datentyps relation angelegt wird, dessen Inhalt auf Datenbank 2 verweist und vice versa.

Wenn man z.B. eine Datenbank mit dem Namen Basdat und ein zweite Datenbank mit den Namen Fre als Beispiel wählt, so stellt sich die Ausgabe dieser Verknüpfung in der folgenden Weise dar:

query name=basdat;part=per
write part=:each[],:fre:each[]
stop 


Kleio wird angewiesen, alles aus der Gruppe per in der Datenbank Basdat auszugeben und anschließend nach Abarbeitung des Übergangs in die Datenbank Fre auch alle in der dortigen Informationsgruppe angetroffenen Daten anzuzeigen. Das Ergebnis hat die folgende Form:


per (8 = "DOK050903")
          idd          DOK050903
          name         ACKERL
          vorname      GOTTHILF
          geschlecht   maennlich
          stand        LED
          anatbb       <anatbb>a (220 = "anatbb-220") : a (220 = "anatbb-220") -> basdatanatbb
          fre          <fre>fam (2120 = "FRE-57-11") : fam (2120 = "FRE-57-11") -> basdatfre
fam (2120 = "FRE-57-11")
          id           FRE-57-11
          basdatfre    <basdat>per (8 = "DOK050903") : per (8 = "DOK05
0903") -> fre
        
<


Ausgehend von Dokument 8 in Basdat (DOK050903) gibt es einen Verweis auf Dokument 2120 (FRE-57-11) in Fre. Dort steht die Angabe FRE-57-11. Diese Form der Ausgabe ist die Standardausgabeform und entspricht der expliziten Einstellung bridge=yes. Es fällt auf, daß auch diejenigen Verweise, deren Inhalt nicht angefordert wurde, detailliert protokolliert werden. In unserem Falle der Verweis "anatbb".

Verändert man nun die obige Aufgabe durch den Parameter bridge=no, so erhält man zwar die inhaltlich identische Ausgabe, doch werden die Verbindungen zu anderen Datenbanken dagegen ohne weitere Angaben dargestellt.  Daher entfällt auch die ausgiebige Protokollierung des Rückverweises von FRE aus Basdat.

query name=basdat;part=per
write part=:each[],:fre:each[];bridge=no
stop
