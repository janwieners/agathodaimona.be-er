Der Parameter continue:

Der Parameter continue erwartet die Angaben yes oder no.  Yes ist das Standardverhalten des Systems. Der Parameter bezieht sich auf die Darstellung von Verknüpfungen  die mittels des Datentyps relation erzeugt wurden.

Man nehme die folgende Aufgabe:

query name=basdat;part=per
write  part=:name,:fre/m:name
stop 

Es wird hier aus der Datenbank "basdat" das Element :name ausgegeben, gefolgt von der Ausgabe des Elements :name aus der Datenbank "fre", die durch das Element :fre des Typs relation angesprochen wird.

Standardmäßig wird hier folgende Ausgabe erzeugt:

per (2 = "DOK050295") : name         ABELE  
> per (2 = "DOK050295") :  ---> <fre>fam (3711 = "FRE-60-27") : > >
fam (3711 = "FRE-60-27") : name         Abele  
<   

Die Verbindung zwischen beiden Datenbanken wird durch die Symbolik der Pfeile angedeutet. Durch Angabe von continue=no wird dieses Ausgabeschema unterdrückt. Es werden nur mehr die Daten aus beiden Datenbanken ausgegeben:

query name=basdat;part=per
write  part=:name,:fre/m:name;cont=no
stop 

per (2 = "DOK050295") : name         ABELE  
fam (3711 = "FRE-60-27") : name         Abele
