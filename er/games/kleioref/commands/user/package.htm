Der Parameter package

Der Parameter package erwartet als Angabe den Namen eines Kleio packages, das durch ein entsprechendes item-Kommando in die Ulwelt eingeführt worden sein muß.

Es folgt ein Beispiel eines packages für den Mailversand bei der dynamischen Kontoverwaltung.
Eine ausführliche Beschreibung der logischen Klasse package und seiner einzelnen Parameter findet sich hier: package

item name=mailsettings;usage=package;overwrite=yes
fields write="mailcopies";
start="Ihre Benutzungsberechtigung für Prometheus läuft ab";
before="Ihre Benutzungsberechtigung für Prometheus läuft ab am:";
after="Ihre Benutzungsberechtigung für Prometheus ist abgelaufen am:";
more="Verständigen Sie Ihren Dozenten.\nIhr\nPrometheus-Team";
connect="/usr/sbin/sendmail"
exit name=mailsettings

Folgende Parameter beeinflussen den Mailversand:

  • connect="/usr/sbin/sendmail" (entspricht dem Default) zu verwendender Mailer,

  • write="mailcopies" (Default: "versandteMails") ein Directory, in das eine Kopie aller versandten mails mit Dateinamen username.mail geschrieben werden.

Aus den restlichen Parametern wird die Mailaufgebaut:

  • start= Subject der email.

  • before= Text, der eingesetzt wird wenn das Ablaufen der Userid noch bevorsteht.

  • after= Text, der eingesetzt wird wenn die Userid bereits abgelaufen ist.

  • more= Zusätzlicher Ratschlag.
