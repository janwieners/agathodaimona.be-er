Der Parameter complete


Der Parameter complete erwartet als Parameterwert den Namen einer temporären Datei, die als Konstante, also in Hochkommata angegeben werden muß. complete erlaubt in Verbindung mit den Parametern suspend und augment einen Katalog anzulegen, der sich auf mehr als eine Datenbank bezieht. Dabei muß complete sich auf eine temporäre Datei beziehen, die zuvor mit suspend angelegt worden ist und durch augment erweitert worden sein kann. Die genannten Parameter funktionieren analog zu ihren Pendants im index-Befehl, weshalb die folgende Erklärung sich kurz fassen kann.

Complete schließt eine durch suspend angelegte temporäre Datei ab. Das heißt, die abgelegten Informationen werden in eine Systemdatei übernommen und die temporäre Datei selbst gelöscht. Dabei ist es für die Funktionweise unerheblich, ob sich dem cataologue-Befehl, der den Parameter suspend enthält, direkt der Befehl mit dem complete-Parameter anschließt oder ob die temporäre Datei dazwischen durch weitere catalogue-Befehle mit dem Parameter augment erweitert wurde.

Ein Beispiel ist die folgende Aufgabe:

query name=sdkuenstler;part=:kuenstlerbez
catalogue name=kuenstler;part=:kuenstlerbez;overwrite=yes;
          suspend="dummy"
continue
query name=dmr;part=:kuenstler
catalogue name=kuenstler;part=:kuenstler;
          augment="dummy"
continue
query name=muenchen;part=:kuenstler
catalogue name=kuenstler;part=:kuenstler;
          augment="dummy"
continue
query name=uustb;part=/aob30
catalogue name=kuenstler;part=:a3100;
          complete="dummy"

Der Inhalt des Feldes "kuenstlerbez" in der Datenbank "sdkuenstler" wird hier in die temporäre Datei "dummy" geschrieben. Dann folgen analog durch Angabe von augment die entsprechenden Inhalte der Datenbanken "dmr" und "muenchen". Schließlich wird  auch noch der Inhalt der Datenbank "uustb" eingetragen und dabei der Gesamtkatalog erstellt. Mit Abarbeitung des Parameters complete im letzten Kommando wird die temporäre Datei (hier: dummy) automatisch gelöscht.
