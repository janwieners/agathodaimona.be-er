Der Parameter suspend


Der Parameter suspend erwartet als Parameterwert den Namen einer temporären Datei, die als Konstante, also in Hochkommata angegeben werden muß. Suspend erlaubt in Verbindung mit den Parametern augment und complete einen Katalog anzulegen, der sich auf mehr als eine Datenbank bezieht. Die genannten Parameter funktionieren analog zu ihren Pendants im index-Befehl, weshalb die folgende Erklärung sich kurz fassen kann.

Suspend legt eine temporäre Datei an, in der die für einen Katalog ausgewählten Informationen einer Datenbank abgelegt werden. Es können sich nun weitere catalogue-Kommandos anschließen, die dann den Parameter augment aufweisen, der auf dieselbe temporäre Datei verweist. Entsprechend wird auch die in die Kommandos ausgewählte Information dort abgelegt. Schließlich wird durch ein letztes catalogue-Kommando mit Einsatz des Parameters complete, der wiederum die temporäre Date benennen muß, der gesamte Vorgang abgeschlossen. es wird nun ein Gesamtkatalog angelegt, der die ausgewählte Gesamtinformation aller beteiligten catalogue-Kommandos enthält.

Ein Beispiel ist die folgende Aufgabe:

query name=sdkuenstler;part=:kuenstlerbez
catalogue name=kuenstler;part=:kuenstlerbez;overwrite=yes;
          suspend="dummy"
continue
query name=dmr;part=:kuenstler
catalogue name=kuenstler;part=:kuenstler;
          augment="dummy"
continue
query name=muenchen;part=:kuenstler
catalogue name=kuenstler;part=:kuenstler;
          augment="dummy"
continue
query name=uustb;part=/aob30
catalogue name=kuenstler;part=:a3100;
          complete="dummy"

Der Inhalt des Feldes "kuenstlerbez" in der Datenbank "sdkuenstler" wird hier in die temporäre Datei "dummy" geschrieben. Dann folgen analog die entsprechenden Inhalte der Datenbanken "dmr" und "muenchen". Schließlich wird  auch noch der Inhalt der Datenbank "uustb" eingetragen und dabei der Gesamtkatalog erstellt. Mit Abarbeitung des Parameters complete im letzten Kommando wird die temporäre Datei (hier: dummy) automatisch gelöscht.
