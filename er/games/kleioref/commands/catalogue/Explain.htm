Der Parameter explain


Der Parameter explain erwartet als Parameterwert eine Konstante, die folglich in Hochkommata angegeben werden muß.

Der Parameter erfüllt eine ganz spezifische Funktion. Der Name eines Katalogs kann durch den Befehl describe wiedergegeben werden. Dies ist z.B. von zentraler Bedeutung, wenn  im Rahmen eines CGI-Servers dem Endanwender eine Auswahl aus verschiedenen Katalogen angeboten werden soll. Dazu müssen die Namen der vorhandenen Kataloge "ausgelesen"  und in einem Web-Formular zur Auswahl angeboten werden. Standardmäßig wird hier der Name des Katalogs ausgegeben, der im name-Parameter des catalogue-Befehls angegeben wurde. 

Dies kann im Einzelfall als ungeeignet erscheinen, wenn man z.B. den Namen dem Katalogs eine Erklärung beifügen möchte. Hierzu dient der Parameter explain. Explain erlaubt, dem Katalog eine frei wählbare Zeichenkette zuweisen, die alternativ zum eigentlichen Namen des Katalogs angezeigt werden kann. Man verändert damit gleichsam den Phänotypus des Katalogs ohne seine tatsächliche systeminterne Benennung zu verändern.

Ein Beispiel mag dies verdeutlichen. Folgendes catalogue-Kommando legt einen Katalog mit dem internen Namen "fundort" an, der in der Darstellung als "Fundort (nur Archäologie)" erscheinen soll:

query name=archgiessen;part=:p_Bildfreigabe="Prometheus"
catalogue name=fundort;part=:p_f_Ort|+:form[", "]|+:p_f_Land;symbol=*;type=terms;overwrite=yes;
      explain="Fundort (nur Arch&auml;ologie)"
continue

Das heißt,  dem Katalog "fundort" ist die Zusatzinformation "Fundort (nur Archäologie)" zugeordnet  worden, die nun mittels eines entsprechenden describe-Komandos abgefragt werden kann, das zugleich alle sonstigen Kataloge des Environments ausgibt:

describe usage=catalogue;write=cgi;first=fundort;display=regular
