Die Direktive block

Die Direktive block dient dazu, innerhalb einer Umgebung, die durch den switch-Befehl definiert wird, einzelne bedingte Kommandos zu definieren und aufzurufen. Die Direktive block kennt die Parameter name und part.

Der Parameter name erwartet als Parameterwert eine frei wählbare Zeichenkette und bestimmt einen frei wählbaren Namen unter dem der jeweilige Kommandoblock in die switch-Umgebung eingeführt wird. Der Block muß durch eine entsprechende exit-Direktive abgeschlossen werden, deren name-Parameter wiederum denselben Namen verwendet.

Der Parameter part erwartet als Parameterwert eine Konstante, die in Hochkommata angegeben werden muß. Der Wert der Konstante gilt als Vergleichsmuster zum Inhalt des Elements, das im part-Parameter des switch-Befehls angegeben wird. Wenn der Inhalt des Elements und das Vergleichsmuster übereinstimmen, gilt die Bedingung als erfüllt und der entsprechende Kommandoblock wird ausgeführt.

Dies geschieht also in folgender Form:

query name=kleio;part=/catalogue[kuenstler,complete,"picasso"]
switch part=:database[];name=selectDB
block name=sdkuenstler;part="sdkuenstler"
exit name=sdkuenstler
write part=:package[:bildnummer,paket1],
block name=piranesi;part="piranesi"
write part=:package[:conversion[:bildreferenz,noending],paket2],
exit name=piranesi
block name=dadaweb;part="dadaweb"
write part=:package[:zzzscan,paket3],
exit name=dadaweb
exit name=selectDB


Es wird also abgeprüft, aus welcher Datenbank der Eintrag "picasso" im Katalog "kuenstler" stammt. Wenn :database[] z.B. den Wert "dadaweb" zurückgibt, so stimmt dieser Wert mit der Konstante im dritten Kommandoblock überein und der entsprechende Auswertungsbefehl

write part=:package[:zzzscan,paket3]

wird ausgeführt.
