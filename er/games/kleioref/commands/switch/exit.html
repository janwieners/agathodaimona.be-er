Die Direktive exit

Die Direktive exit schließt eine Befehlsumgebung, die durch das Kommando switch eingeführt wurde, ab. Ebenso beendet exit einzelne Kommandoblöcke, die jeweils durch eine block-Direktive eingeführt worden sein müssen. Exit kennt nur den Parameter name. Dieser wiederum erwartet als Parameterwert eine Zeichenkette, die vorab durch den name-Parameter des switch-Befehls oder der block-Direktive eingeführt worden sein muß. 

Typischerweise hat also die Kommandoumwelt des switch-Befehls folgende Grundstruktur:


switch name=selectDB
...
exit name=selectDB


Nicht nur der switch-Befehl selbst führt einen Namen ein, der durch eine exit-Direktive aufgehoben (also abgeschlossen) werden muß. Auch die einzelnen block-Direktiven innerhalb der switch-Umgebung sind durch eindeutige Namen gekennzeichnet. Daraus wiederum folgt, daß die Abfolge der Kommandoblöcke durch den gesamten switch-Befehl geklammert werden muß. Das heißt, die einzelnen Blöcke werden ebenfalls durch name-Parameter (der block-Direktive) identifiziert und innerhalb der switch-Umgebung durch entsprechende exit-Direktiven abgeschlossen bis schließlich die finale exit-Direktive, die sich auf den switch-Befehl bezieht, die Gesamtstruktur beendet. Dies hat folgende Grundform:

switch name=selectDB

block name=block1
...
exit name=block1

block name=block2
...
exit name=block2

block name=block3
...
exit name=block3

exit name=selectDB
