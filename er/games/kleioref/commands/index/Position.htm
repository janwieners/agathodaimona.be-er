Der Parameter position:

Üblicherweise reserviert das System für jeden Spalteneintrag pro Zeile genau so viele Zeichen, wie zur Darstellung des Inhalts zzgl. einer Leerstelle als Spaltentrenner notwendig sind. Daraus folgt, daß bei Daten unterschiedlicher Länge die Registerzeilen unterschiedliche breit sind und die jeweiligen Spalten daher  nicht ausgerichtet erscheinen.

Der Parameter position ändert dieses Systemverhalten. Durch Angabe des Parameterwerts yes (no ist Standardverhalten) wird das System angewiesen, die Spalten gleichmäßig auszurichten. dabei gelten folgende Regeln: Bei einer Standardbildschirmausgabebreite von 80 Zeichen (eine andere Seitenbreite kann durch denn signs-Parameter in der options-Direktive eingestellt werden)   reserviert das System 13 Zeichen für die standardmäßige Dokumentenidentifikation bzw. jede Spalte des Registers, die durch einen identification-Parameter ausgewählt wurde (i.e. 12 Zeichen plus eine Leerstelle).  Die verbleibende Anzahl der Stellen wird durch die Anzahl der übrigen Spalten dividiert und von diesem Resultat jeweils eine Stelle als Spaltentrenner subtrahiert. Das Ergebnis der Rechenoperation entspricht der Spaltenbreite in der Ausgabe.

So erzeugt folgender Befehl

index part=:name;part=:coll[:vorname];part=:hko;position=yes

ein vierspaltiges Register mit regelmäßiger Spaltenbreite.

Eine weniger pauschale, sondern benutzerpsezifische Einstellung der jeweiligen Spaltenbreiten kann durch den  sign-Parameter erreicht werden.
