Der Parameter result:

Der Parameter result erwartet als Parameterwert ein einzelnes, frei wählbares Zeichen.

Beim Einsatz mehrere Index-Befehle werden die einzelnen Registerzeilen üblicherweise unabhängig von der Nummer des Index-Befehls sortiert, der diese generiert hat. Durch Anwendung der Parameter lines ,  also und source kann dies modifiziert werden.

Dabei gehen die genannten Parameter davon aus, daß jeweils der gesamte Spalteninhalt zur Sortierung herangezogen wird. Im Unterschied dazu erlaubt der Parameter result, daß der Parameterwert zusätzlich als Sortierkriterium eingefügt wird. Bei zusammenhängenden Zeichenketten wird das jeweilige Zeichen an das Ende des Spalteninhalts gesetzt. Wenn der Spalteninhalt durch Leerstellen gegliedert ist, so ersetzt der Parameterwert von result die erste vorkommende Leerstelle. Damit result funktioniert, muß einer der Index-Befehle mit dem Parameter second markiert sein.

Diese Einfügungen gelten jeweils für die Sortierung, nicht aber für die Darstellung der Ausgabe!

Eine entsprechenden Aufgabe hat die folgende Grundform:


query name=basdat;part=per
index part=pdat/part[kar,1]:beruf;second=~;part=:form["Zeile 1"];without=yes
index part=pdat/part[kar,2]:beruf;result=y;part=:form["Zeile 2"];without=yes
index part=pdat/part[kar,3]:beruf;result=x;part=:form["Zeile 3"];without=yes
stop 


Im zweiten und dritten Index-Befehl werden an das Ende der Zeile bzw. anstelle der ersten Leerstelle ein "y" bzw. ein "x" eingefügt. Da "x" vor "y" sortiert wird, erscheinen bei identischem Inhalt diejenigen Daten, die durch den 3. Index-Befehl erzeugt wurden, vor jenen, die der dritte Befehl generiert hat.
