Der Parameter number:

Der Parameter number erwartet als Parameterwert eine ganze Zahl. Der Parameter regelt die Behandlung von sog. "imbedded numbers", also von Zahlen, die in Texten enthalten sind.

Man habe ein Element :datum, das u.a. die Datumsangaben "11. Mai 1800" und "3. Mai 1800" enthält. Für den Rechner sind diese Angaben "nur" Texte.

Daher liefert die Aufgabe

index part=:datum

ein unbefriedigendes Ergebnis:

11. Mai 1800 DOK000015

3. Mai 1800 DOK000009


Der 11. Mai wird vor dem 3. Mai sortiert (eins steht vor drei!). Durch Angabe des Parameters number kann das System nun aber angewiesen werden, die eingebetteten Zahlen für die Sortierung  "nach links" auf eine bestimmte Stellenzahl  mit Nullen aufzufüllen. Die Zahlenangabe bei number entspricht dabei der Zahl der Stellen bis zu der aufgefüllt wird.

Der Befehl

index part=:datum;number=2

weist das System also an, enthaltene Zahlen zweistellig zu deuten. Die Zahl "11" bleibt demnach zweistellig. Dagegen wird die Zahl "3" auf  "03" aufgefüllt. Da "0" vor "1" sortiert wird, und zugleich der Parameter nur die Sortierung, nicht aber das Aussehen der Daten beeinflußt, hat das Register nun folgende Form:

3. Mai 1800 DOK000009

11. Mai 1800 DOK000015
