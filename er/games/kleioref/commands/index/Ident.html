Der Parameter Identification

  Der Parameter identification entspricht in seiner Funktionalität  grundsätzlich dem part -Parameter, unterscheidet sich freilich in einigen wesentlichen Punkten:

  1. Das System behandelt Elemente, die durch den identification-Parameter ausgewählt wurden, als temporäre Dokumentenidentifikation. Das bedeutet, daß das System hinsichtlich der Verwendung der Daten als primary key von einer Spaltenbreite von maximal 12 Zeichen ausgeht.
  2. Daraus folgt weiterhin, daß maximal eine Spalte des Registers durch den identification-Parameter erzeugt werden sollte, während man für die übrigen auf den  part-Parameter zurückgreifen kann.
  3. Nachdem der Inhalt derjenigen Spalte, die durch identification erzeugt wurde, als Dokumentenidentifikation gilt, entfällt die Ausgabe der standardmäßigen Identifikation.
  4. Für Spalten, die durch identification erzeugt wurden, stehen weitere Steuerparameter zur Verfügung. Hier sei besonders auf den sequence-Parameter verwiesen, der der Erzeugung von Belegstellenlisten dient.
