Der Parameter suspend:


Das Ergebnis eines Index-Befehls wird generiert, indem das System entsprechend der Abfrage die Datenbank durchsucht und alle angeforderten Informationen erst einmal in einer temporären Datei sammelt. Nach dem Ende der Suche werden die Informationen in der temporären Datei sortiert und ausgegeben. Dies erfolgt (je nach Einstellung) durch die sortierte Ausgabe auf dem Bildschirm oder die Ablage des Ergebnisses in einer Datei. Nach der Datenausgabe wird die temporäre Datei wieder gelöscht.

Der Parameter suspend erlaubt die Abänderung dieses Mechanismus. Er erwartet als Parameterwert den beliebig wählbaren Namen einer Datei, der in Hochkommata angegeben werden muß. Diese Datei wird bei der Abarbeitung des Index-Befehls nunmehr als temporäre Datei verwendet. Im Unterschied zum Standardverhalten erfolgt bei Einsatz des Parameters suspend keine Ausgabe des Inhalts der temporären Datei. Sie wird vielmehr beibehalten und kann durch weitere Index-Befehle erweitert werden. Damit ist es möglich, den Inhalt mehrerer Datenbankabfragen, sie sich sogar auf unterschiedliche Datenbanken beziehen können, in einer einzigen temporären Datei zu vereinigen. Diese temporäre Datei schließlich kann abschließend wiederum in Form eines einzigen, einheitlichen Registers ausgegeben werden.

Die Erweiterung einer durch suspend angelegten temporären Datei und die schließliche Ausgabe des Listings werden durch die Parameter augment und complete gesteuert.

Ein Beispiel mag den Mechanismus verdeutlichen. Wir haben drei Datenbanken (basdat, fre und frk) in denen jeweils Berufsangaben (:beruf) vorliegen. Ziel soll es sein, aus den drei Datenbanken ein einheitliches Register aller Berufsbelege zu generieren.

Mittels des folgenden Befehls weisen wir das System an, die Berufe in der Datenbank basdat in einer temporären Datei mit dem Namen "datei1" zwischenzuspeichern:

query name=basdat;part=:beruf
index part=:beruf;suspend="datei1"
cont  

Im folgenden Schritt werden wir nun mit dem Parameter augment , auch die Berufsangaben aus der Datenbank fre in die temporäre Datei "datei1" einfügen.
