Der Befehl database:


Der Befehl database dient der Definition, das heißt der Anlage von Datenbanken. Er dient im wesentlichen zwei Zwecken:

  1. Durch den Parameter name wird der jeweiligen Datenbank ein Name zugewiesen, unter dem Sie später angesprochen werden kann.
  2. Der Befehl database eröffnet eine Umgebung, innerhalb derer durch verschiedene Direktiven, die Struktur der Datenbank definiert werden kann. Dementsprechend wird dieser Teil der Datenbankumwelt in der Kleio-Diktion als "Strukturvereinbarung" bezeichnet. Üblicherweise wird die Strukturvereinbarung durch die Direktive exit abgeschlossen.
Der Aufbau der Strukturvereinbarung wird maßgeblich vom Format der Eingabedaten bestimmt. Kleio kann gegenwärtig drei verschiedene Arten von Rohdaten interpretieren. Dies sind zum ersten formal ausgezeichnete ASCII-Daten, die entweder im XML-Format oder auch im klassischen Kleio-Format vorliegen können. Daneben ist Kleio in der Lage, Daten der kommerziellen Softwareprogramme sisis und dbase einzulesen. Die entsprechende Auswahl wird durch den Parameter other festgelegt. Die entsprechende Auswahl des Eingabeformats zieht bei der Definition der Strukturvereinbarung zwingend die Verwendung entsprechender Direktiven für die jeweiligen Eingabeformate nach sich.  Das heißt, daß eine Strukturvereinbarung, die sich auf XML-Daten bezieht andere Direktiven enthält, als eine Vereinbarung für sisis-Daten, die sich wiederum von einer Vereinbarung für das dbase-Format unterscheidet.


Als Beispiel eines gültigen database-Befehls kann folgende Zeile gelten:

database name=basdat;
     first=per;
     overwrite=yes

Es wird hier eine Datenbank mit dem Namen "basdat" angelegt. Die jeweils erste Gruppe eines Dokuments heißt "per". Wenn bereits Systemdateien mit Bezug zu dieser Datenbank bestehen, so werden diese überschrieben. Die zugehörigen Daten müssen müssen (da nichts weiteres angegeben ist) im Format "kleio" vorliegen.
