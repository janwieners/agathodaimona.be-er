Der Parameter usage:


Der Parameter usage steuert die Zugriffsrechte auf die Datenbank. Er erwartet die Parameter readonly und modifiable. Letzterer ist das implizite Standardverhalten des Systems.

Die Angabe des Parameterwerts readonly verhindert, daß der Inhalt der Datenbank verändert werden kann. Dies ist besonders dann von Bedeutung, wenn eine Kleio-Datenbank als Web-Server betrieben wird, da damit jede unbefugte Manipulation der Daten verhindert wird. Es ist aber auch bei Angabe des Parameters readonly möglich, Verwaltungstätigkeiten an der Datenbank durchzuführen. So kann der Administrator z.B. bei laufendem Web-Server einen neuen Katalog anlegen.

Die Umwelt der Datenbank (also z.B. Codebücher und andere Vereinbarungen) sind durch den Parameter usage im database-Befehl nicht betroffen. Es ist aber möglich, auch die Objekte in der Umwelt der Datenbank zu sperren, indem man im Befehl environment den Parameter usage analog zu der hier beschriebenen Weise einsetzt.
