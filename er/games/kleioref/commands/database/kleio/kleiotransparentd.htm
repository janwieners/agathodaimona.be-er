Die Direktive transparent:


Die Direktive transparent ist sinnvoll bei der Verarbeitung von Daten, die im XML-Format vorliegen. Sie erlaubt bestimmte Tags, Inhalte bestimmter Tags und/oder Attribute zu ignorieren, indem diese als "tranparente Symbole" derfiniert werden. Das heißt, daß das System diese Tags und (fall erwünscht) auch deren  Inhalt bei der Kompilation der Datenbank übergeht. Damit ist ein selektiver Zugriff auf die Daten in der XML-Datei möglich. Der Name des betreffenden Tags muß durch den Parameter name angegeben werden, der sich jeweils zugleich auf das Start- und das Endtag bezieht. 

Man benötigt diese Anweisung z.B., wenn ein XML-Dump aus einem anderen Datenbankprogramm verarbeitet werden soll, der neben den eigentlichen Daten u.U. auch systeminterne Angaben enthalten kann, die durch entsprechende Auszeichnungen kenntlich gemacht sind. Diese können durch die Direktive transparent als irrelevant gekennzeichnet werden. Das System kann diese dann beim Einlesevorgang gezielt übergehen.

Ein Beispiel für eine korrekte transparent-Direktive stellt das folgende Beispiel dar:

transparent name=text;type=element

Hier wird das System angewiesen, in einem Datensatz das Startag "<text>" bzw. das Endtag "</text> zu ignorieren. Der Parameter type mit dem Wert Element legt aber fest, der Inhalt des Tags dennoch in die Datenbank übernommen wird. Durch andere Parameterwerte bei type kann dieses Verhalten differenziert modifiziert werden.
