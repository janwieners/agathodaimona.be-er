Die Direktive property:


Die Direktive Property wird bei Eingabedaten eingesetzt, die im XML-Format vorliegen. Sie weist bestimmten Gruppen eine Eigenschaft zu, indem sie bestimmte Elemente innerhalb einer Gruppe dergestalt definiert, daß sie nicht Bestandteil der Datenbank sind, sondern als Eigenschaft der Gruppe und der in ihr enthaltenen Elemente gelten. Das heißt, daß die mittels property definierten Elemente (=Tags oder Attribute in den XML-Quelldaten) nicht genuiner Teil der Datenbank sind, sondern unabhängig von der Struktur der jeweiligen Datenbank lediglich mit dieser verbunden sind. 

Der grundsätzliche Sinn der Direktive property besteht darin, Elementen oder Gruppen einer Datenbank unabhägig von der genuinen Datenbankstruktur allgemeine Informationen zuzuordnen.

Ein Beispiel mag dies verdeutlichen. Gegeben sei ein Auszug aus einer Handschriftenbeschreibung im XML-Format, der bibliographische Angaben umfaßt. Unter anderem seien dies die folgenden Daten:

Wie man erkennen kann, enthält der Tag <listBibl> ein Attribut des Namens "authority", das den Wert " Hollas: 150-151 " aufweist. Inhaltlich besagt dies ungefähr, daß die nachfolgenden bibliographischen Angaben (Tag <bibl> ) sich ihrerseits auf die Literaturangabe "Hollas: 150-151" stützen. Es stünde dem Benutzer prinzipiell frei, das Attribut "authority" als Element innerhalb der Gruppe listBibl zu definieren. In unserem Falle führen wir das Attribut jedoch nicht mittels der Direktive element , sondern der Direktive property in das Environment ein:
property name=authority;
     association=version
Damit erreichen wir nun, daß die Eigenschaft authority="Hollas: 150-151" für der Gruppe listBibl aus unserem obigen Datenbeispiel zugeordnet wird. Der Parameter association=version legt weiter  fest, daß die betreffende Eigenschaft nicht nur auf  alle Elemente der Gruppe, sondern auch auf alle von dieser abhängigen Gruppen und deren Elemente vererbt wird.

Es ist nun zum Beispiel möglich, den Inhalt der Informationsgruppe bibl auszugeben, die den Inhalt des Tags <bibl > repräsentiert, den wir als das Element biblitem definiert haben.

Ein einfacher write-Befehl hier würde dies folgendermaßen aussehen:

query name=ceec;part=bibl
write part=:biblitem
stop


und folgendes Ergebnis liefern:

cec (1 = "kn28-0083ii") : biblitem     Hartzheim , p. 46-47, 53, 55, 139
cec (1 = "kn28-0083ii") : biblitem     Jaffe/Wattenbach, p. 29-31
cec (1 = "kn28-0083ii") : biblitem     C. H. Beeson, lsidor-Studien (Munich 1913), p. 70, 77, 90, 95, 102, 128

                                                                  . . .

cec (1 = "kn28-0083ii") : biblitem     E. Zinner, Verzeichnis der astronomischen Handschriften des deutschen Kulturgebiet (Munich
                                        1925) (text an ff. 146r-171v as no. 705)
cec (1 = "kn28-0083ii") : biblitem     see also CLA Suppl. no. 1154, p. 62.


Nun haben wir jedoch (wie oben dargestellt) zusätzlich das Attribut authority  mittels der Direktive property eingeführt und damit seinen Inhalt, nämlich die Information "Hollas: 150-151 " mit der Datenbank verbunden. Es ist also möglich, diese Eigenschaft gemeinsam mit dem durch sie charakterisierten Element auszugeben. Dazu bedienen wir uns der Elementarfunktion : version[] , die die Property-Eigenschaften eines Elements anzeigt.

Entsprechend modifizieren wir die obige Aufgabe:


query name=ceec;part=bibl
write part=:biblitem,:version[:biblitem]
stop

und erhalten nun folgendes Ergebnis:

cec (1 = "kn28-0083ii") : biblitem     Hartzheim , p. 46-47, 53, 55, 139
cec (1 = "kn28-0083ii") : authority    Hollas: 150-151
cec (1 = "kn28-0083ii") : biblitem     Jaffe/Wattenbach, p. 29-31
cec (1 = "kn28-0083ii") : authority    Hollas: 150-151
cec (1 = "kn28-0083ii") : biblitem     C. H. Beeson, lsidor-Studien (Munich 1913), p. 70, 77, 90, 95, 102, 128
cec (1 = "kn28-0083ii") : authority    Hollas: 150-151

                                                                        . . .

cec (1 = "kn28-0083ii") : biblitem     E. Zinner, Verzeichnis der astronomischen Handschriften des deutschen Kulturgebiet (Munich
                                        1925) (text an ff. 146r-171v as no. 705)
cec (1 = "kn28-0083ii") : authority    Hollas: 150-151
cec (1 = "kn28-0083ii") : biblitem     see also CLA Suppl. no. 1154, p. 62.
cec (1 = "kn28-0083ii") : authority    Hollas: 150-151

Die Angabe "Hollas: 150-151" wird nun als Eigenschaft des Elements "biblitem" in der Gruppe bibl sichtbar. Entsprechend erscheint diese Information bei jeder Ausgabe des Elements auf, dessen Inhalt solchermaßen eine nähere Qualifizierung erfährt.

Grundsätzlich gilt, daß bei Angabe des Parameters association mit dem Wert version alle Eigenschaften, die durch property eingeführt wurden, auf alle nachrangigen Gruppen und die darin enthaltenen Elemente vererbt werden. Damit können Elemente, die in der Hierarchie weit "unten" angesiedelt sind, ggf. eine größere Anzahl von zugeordneten Eigenschaften aufweisen, die sie jeweils von allen übergeordneten Gruppen ererbt haben.

Dabei können jede Gruppe und jedes Element grundsätzlich eine beliebige Anzahl von Property-Eigenschaften besitzen. Es gilt jedoch folgende Regel: Wenn ein Element oder eine Gruppe eine Property-Eigenschaft besitzen und zugleich eine gleichnamige Property-Eigenschaft von einer übergeordneten Gruppe ererben, so überschreibt der Inhalt der genuinen Property-Eigenschaft den Inhalt der ererbten Eigenschaft.

Durch die Funktion :view[] können ggf. alle Property-Eigenschaften innerhalb einer Gruppe angezeigt werden. Die Gruppenfunktion preference[] bzw. die Elementarfunktion :preference[] erlauben die gezielte Ausgabe von Gruppen bzw. Elementen entsprechend dem Inhalt ihrer Propertys.
