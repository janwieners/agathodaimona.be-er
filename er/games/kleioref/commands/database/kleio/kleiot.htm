Grundsätzlich gilt, daß das Format "kleio" zum einen das Handling des "klassischen" Kleio Eingabeformats betrifft, dessen Aussehen sich wie folgt darstellt:


PER$DOK030011/ABELE/CARL/M/KIRCHHEIM,TECK/OA KIRCHHEIM
PDAT$
KAR$1.7.1846-1.7.1850/FABRIKANT,SESSEL(CARL ABELE)#WSP
MOB$ 
WSP$DOK030011/ABELE/CARL/FABRIKANT,SESSEL//KIRCHHEIM,TECK
/OA KIRCHHEIM/1/1.7.1846/1.7.1850#AB 50 FORT/BEM=UNSICHERE LESUNG

Hier wird die Eingabe der Daten durch sog. "Datensignalzeichen" ($/#%?=<>;:) gesteuert. Der Name der Informationsgruppen steht vor dem "$"-Zeichen, Elemente werden entsprechend der Reihenfolge, die in der part-Dirketive festgelegt ist, durch "/" getrennt aufgeführt. Kommentare und Originaltextfelder werden durch "#" bzw. "%" eingeleitet. Abweichend von de Vorgaben der Strukturverinbarung können Elemente durch Etiketten eingeführt werden. Für Einzelheiten sei auf Kapitel 4 des Systemhandbuchs verwiesen.

Alternativ dazu umfaßt das Datenformat "kleio" auch die Handhabung von XML-Dateien. Kleio benötigt dazu keine DTD. Die reinen Eingabedaten sind hinlänglich. Da viele Datenbankprogramme in der Lage sind, einen Datendump im XML-Format zu erzeugen, kann dieses Format auch dazu benutzt werden, Daten anderer Softwareprodukte als Kleio-Datenbank einzulesen.

Die Unterscheidung zwischen der Verarbeitung des "klassischen" Kleio-Formats und einer XML-Datei wird im other-Parameter des read-Befehls getroffen. Als Standard ist dort das "klassische" Format eingestellt, wogegen die Einstellung "other=markup" die Verarbeitung einer XML-Datei anordnet.
