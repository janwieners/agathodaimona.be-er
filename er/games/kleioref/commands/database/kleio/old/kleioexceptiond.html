Die Direktive exception:

Durch die Direktive element können einem Element bestimmte Eigenschaften zugewiesen werden. So können u.a. der Datentyp (text [Standard], date, number, category, relation, location oder image) des Elements und seiner Aspekte (Kommentar  und Originaltext) festgelegt werden. Weiterhin kann der Nutzer entscheiden, ob bei Mehrfacheinträgen die Aspekte dem Gesamteintrag oder einem Teileintrag untergeordnet sein sollen. Ebenso kann durch die Direktive identification einem Element die Eigenschaft der Dokumentenidentifikation zugewiesen werden.

Die entsprechenden Festlegungen sind Bestandteil der Programmumwelt. Das heißt, daß ein Element desselben Namens in verschiedenen Datenbanken für gewöhnlich dieselben Eigenschaften aufweist, sofern die betreffenden Datenbanken derselben Umwelt angehören. Dies kann in bestimmten Fällen zu Problemen führen, wenn es aus verschiedenen Gründen wünschenswert oder sogar notwendig erscheint, daß Elemente desselben Namens in unterschiedlichen Datenbanken unterschiedliche Eigenschaften aufweisen.

In diesem Falle kann durch die Direktive exception ausdrücklich angeordnet werden, daß einem bestimmten Element im Kontext einer bestimmten Datenbank ausnahmsweise eine Eigenschaft zugewiesen sein soll, die von der Globaldefinition in der Umwelt abweicht. Dabei können grundsätzlich alle oben genannten Eigenschaft eines Elements geändert werden.

Das heißt, daß prinzipiell sowohl der type-, der first-, der second- und der order-Parameter der Direktive element im entsprechenden Sinn auch bei exception eingesetzt werden kann. Im gegenwärtigen Zustand ist die Implementierung dieser Parameter für exception noch nicht voll abgeschlossen. Jedoch steht der Parameter identification bereits voll zur Verfügung.
