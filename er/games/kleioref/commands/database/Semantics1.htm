Der Parameter semantics:

Der Parameter semantics erwartet die Parameterwerte yes und no. Letzteres ist das Standardverhalten des Systems. Der Parameter steuert die Löschung von Datenbasen. Wird ein database-Befehl ausgeführt und dabei der Parameter overwrite mit dem Parameterwert yes angegeben, so löscht Kleio ggf. den Inhalt einer bestehenden Datenbank. Allerdings bleiben in der Umgebung der Datenbank die Definitionen für die Gruppen und Elemente  der Datenbank erhalten.

Dies kann zu unangenehmen  Folgen führen. Wird etwa in einer Datenbank mit dem Namen "Datenbank1" eine Element "nummer" definiert und diesem der Datentyp number zugewiesen, so bleibt diese Definition auch dann gültig, wenn der Inhalt von "Datenbank1" durch Ausführung des Befehls database und Angabe von overwrite=yes gelöscht wird. Wenn nun eine weitere Datenbank erstellt wird, in der ebenfalls ein Element des Namens "nummer" vorgesehen ist, so setzt Kleio für dieses Element denselben Datentyp voraus. Ein ähnliches Problem stellt sich, wenn in "Datenbank1" eine bestimmte Gruppe definiert wird, deren Name in weiteren Datenbanken Verwendung finden soll. Kleio erwartet in diesem Fall, daß die Gruppen jeweils entsprechend aufgebaut sind.

Anders ist das Systemverhalten, wenn eine Datenbank mit overwrite=yes und zugleich mit semantics=yes überschrieben wird. In diesem Falle löscht Kleio nicht nur den Inhalt der betreffenden Datenbank, sondern entfernt auch alle definitorischen Angaben zu Elementen und Gruppen aus der Umwelt. Damit können die betreffenden Elemente oder Gruppe in einer anderen Datenbank frei verwendet werden.

Ein entsprechender database-Befehl muß daher der folgenden Form aufweisen:

database name=datenbank1;first=p;overwrite=yes;semantics=yes
